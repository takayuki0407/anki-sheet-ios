# セッション開始

新しいセッションを開始します。前回までの状態を復元するため、以下を順に実行してください。

## Step 1: 状態ファイルを読む

以下を順に読み込み、プロジェクトの現状を把握する：

1. **CLAUDE.md** — 設計・方針・アーキテクチャ決定事項・未解決の課題
2. **docs/tasks.md** — 未完了タスクと優先順位
3. **HANDOFF.md**（存在すれば） — ライブな作業状態（Build番号・審査状況・次の一手・未pushコミット）
4. **docs/research/**（関連ファイルがあれば） — 過去の調査結果

## Step 2: 現状を報告

以下の形式で日本語報告する：

```
【セッション開始】

📍 現在地
[HANDOFF.md / tasks.md から、今どこまで進んでいるか]

✅ 直近の完了
[前回 wrap-up の「セッション履歴」最新分から]

📋 今回の優先タスク
1. [最優先]
2. [次点]
3. [次点]

⚠️ 未解決の課題（着手前に意識すべきもの）
- [CLAUDE.md の「未解決の課題」から関連するもの。なければ「なし」]

❓ 着手前に確認したいこと
- [曖昧な点・判断が必要な点があれば。なければ「なし」]
```

## Step 3: 着手前の確認

- 状態ファイルの記述と実際のコード/リポジトリ状態に食い違いがないか、軽く確認する
- 食い違いがあれば報告する（外部化が不足しているサインなので、その場で補強提案する）
- ユーザーの指示を待ってから実作業に入る（勝手にコードを書き始めない）
